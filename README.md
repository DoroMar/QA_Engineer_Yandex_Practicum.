# Инженер по тестированию Яндекс Практикум
В этом портфолио собраны проекты, выполненные во время обучения по специальности Инженер по тестированию в Яндекс.Практикуме.

### Спринт 1 Проектирование тестов

* Постановка задач на тестирование, тест-анализ и тест-дизайн, работа с Figma и тестовым стендом сервиса Яндекс.Маршруты. 
* Освоение алгоритмов тестирования, подготовка тестового набора и проверка валидации полей ввода времени и адресов. 
* Составление mind-map и блок-схем логики расчета времени и стоимости маршрута, а также разработка тест-кейсов для проверки расчета времени и стоимости поездки на собственном автомобиле. 
* Дополнительно: тестирование формы авторизации (на примере Kittygram) и формы обратной связи.

### Спринт 2 Тестирование веб-приложений

* Проверка интерфейсов, адаптивности, кроссплатформенности и кроссбраузерной совместимости с использованием DevTools. 
* Применение техник тест-дизайна (позитивное/негативное тестирование, таблицы принятия решений, попарное тестирование) и работа в YouTrack с оформлением баг-репортов.
* Функциональное и UI-тестирование каршеринга сервиса Яндекс.Маршруты: чек-листы на вёрстку и навигационную карту, проверка логики окон «Способ оплаты» и «Добавление карты», тест-кейсы на кнопку «Забронировать», тестирование формы выбора тарифного плана.

### Спринт 3 Тестирование мобильных приложений 

* Жизненный цикл разработки, модели разработки и этапы тестирования, особенности требований и функционирования мобильных приложений. 
* Работа с физическими устройствами и эмуляторами, использование Android Studio и анализ логов.
* Функциональное и регрессионное тестирование мобильного приложения Яндекс Метро на Android: составление чек-листов, тестирование в эмуляторе Android Studio и оформление баг-репортов по результатам проверки.

### Спринт 4 Тестирование API

* Работа с JSON, XML, SOAP, WSDL, документацией (Swagger, Apidoc) и логами. 
* Использование Postman (динамические URL, cURL, генерация запросов) и базовые знания БД.
* Применение различных видов тестирования (ручное и автоматизированное, функциональное и нефункциональное, регрессионное, смоук, компонентное, интеграционное, системное, исследовательское, чёрный/белый/серый ящик).
* Составление чек-листа на новую функциональность API сервиса Яндекс.Прилавок, проведение тестирования, оформление баг-репортов и подготовка отчёта о тестировании.

### Спринт 5 Основы баз данных

* Введение в базы данных и работу с консолью, создание срезов данных и использование агрегирующих функций в SQL.
* Определение взаимоотношений между таблицами и применение различных типов объединений (JOIN).
* Анализ сетевого трафика с использованием Charles.
